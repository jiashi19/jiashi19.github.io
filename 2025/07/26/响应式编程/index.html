响应式编程

响应式编程

响应流

  • 响应流必须是无阻塞的。
  • 响应流必须是一个数据流。
  • 它必须可以异步执行。
  • 并且它也应该能够处理背压。

背压

生产者可以感受到消费者反馈的消费压力，并根据压力进行动态调整生产速率。

具体：消费者在订阅后会向生产者发送一个request(n)信号，表示希望接收n个元素。

Publisher和Subscriber

由于响应流的特点，我们不能再返回一个简单的POJO对象来表示结果了。必须返回一个类似Java中的Future的概念，在有结果可用时通知消费者进行消费响应。

publisher特性：

  • 当没有订阅时发布者什么也不做。
  • 数据是推送（Push）给消费者的，以区别于平时我们的拉（Pull）模式。
  • 只有在有订阅（subscribe()）时才会开始发送数据。

Reactor 中的常用操作符

在 Java 的 Project Reactor（Spring WebFlux 的基础）中，响应流由以下两种基本类型组成：

  • Mono<T>：表示 0 或 1 个元素的异步结果。
  • Flux<T>：表示 0 到 N 个元素的异步流。

groupby.().flatmap() 对每组进行独立异步处理

.subscribe和.map（.flatmap）:前者订阅并触发数据流，无return，后者是必须有return

flatMap的转换Function要求返回一个Publisher，这个Publisher代表一个作用于元素的异步的转换操作；而map仅仅是同步的元素转换操作。

**.block()**将异步操作转换成同步操作，例如将Mono 转换成 Student，去掉外面的Mono<>，但会产生warning，因为破坏了反应式的非阻塞模型，不推荐该写法。
